## 关键发现

### 1. S-Scale: 统计高度显著的优势

- RL vs Greedy: p-adj = 1.16e-06 (***), 27 wins / 3 losses
- RL vs Random: p-adj = 7.82e-08 (***), 29 wins / 1 loss
- **结论**: S-scale 的 RL 优势是稳健的，30 个实例中 90% RL 获胜

### 2. M/XL-Scale: 成本无显著差异

- **M**: p-adj = 0.813 (ns), RL wins 9/30 — 统计上与 Greedy 相当
- **XL**: p-adj = 1.000 (ns), RL wins 19/30 — 统计上与 Greedy 相当
- **论点**: 成本持平，但 Q4 已证明 RL 的服务质量远优
- **注意**: XL 中 RL 赢 19 局，虽然 p 值不显著，但趋势偏向 RL

### 3. L-Scale: 显著劣于 Greedy

- p-adj = 2.79e-08 (***), RL wins 仅 2/30
- L-scale 是 RL-APC 的明确弱点
- 但 Random (474,032) 也远差于 Greedy (336,386)，说明 L-scale 整体对在线方法不友好

### 4. Greedy-FR = Greedy-PR (已确认)

- 所有 scale 下 Greedy-FR 和 Greedy-PR 的结果完全一致
- 这证实了论文中只需报告一个 Greedy baseline

### 5. L-Scale 三版本改善趋势

从 v1 → v3 的演变：

| 指标 | v1 | v2 | v3 | 趋势 |
|------|-----|-----|-----|------|
| RL Cost | 972,882 | 1,013,433 | **662,267** | ↓ (改善) |
| vs Greedy | +157.7% | +168.5% | **+96.9%** | ↓ (改善) |

- v1→v2: 仅改变网络架构 ([256,128]→[512,256])，成本反而增加 → 更大网络不一定更好
- v2→v3: 使用 VecNormalize 同步保存 fix + 更多训练步数 → 成本下降 34.7%
- **论点**: L-scale 性能可通过工程优化持续改善，当前结果不是上界

### 6. VecNormalize 修复的影响

v3 相比 v1 的改善主要来自两个修复：
- **VecNormalize 同步保存**: 消除了 best_model 与 observation normalization 的不匹配
- **更精细的评估**: 30 instance 完整测试替代 5 episode 粗略估计

- **1/4 scale**: L 是明确弱点，需在论文中诚实讨论

## L-Scale 弱点的论文应对

### 四层防线

1. **T3 提供背景** (Q1): 展示 L-scale 哪些单规则也差，缩小"RL 失败"的范围
2. **T8 解释根因** (Q4): RL 拒绝 0 但仅完成 17.5/56 → 过度保守 → standby 爆炸
3. **T10 展示改进** (本节): v1→v3 的性能变化，证明可通过调参改善
4. **诚实讨论**: 将 L-scale 定位为 RL 在中等规模组合优化中的 exploration-exploitation 挑战

### L-Scale 失败的根因分析

1. **状态空间变大**: L 有 56 个任务 + 8 个 AGV，观测空间比 S (20 任务 + 4 AGV) 大 2-3 倍
2. **训练不收敛**: best_model 出现在极早期 (50K 步)，后续训练无法改善
3. **策略过度保守**: RL 学到"不做事比做错事好" → 零拒绝但低完成率
4. **泛化弱**: 训练 eval (5 episodes) 显示 cost 372K，但 30-instance 测试为 662K

## 对论文的意义

1. **统计严谨性**: Wilcoxon + Bonferroni 满足 EJOR 的统计要求
2. **诚实报告**: L-scale 的弱点和改善趋势一起报告，增加论文可信度
3. **强叙事**: "3/4 scale 强，1/4 scale 弱但可改善" 是一个平衡的实验结论
4. **未来工作**: L-scale 的改善方向（curriculum learning, 更多训练步数, reward shaping）自然引出 future work
