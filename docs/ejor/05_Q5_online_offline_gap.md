# Q5: 与离线最优差多远？

> **结论: 所有在线方法与离线最优的 gap 均 >370%，这是在线决策问题的固有代价，而非算法缺陷。RL-APC 和 Greedy-FR 的 gap 相当，说明两者都在在线约束下达到了相似的性能天花板。**

## 关键发现

### 1. Gap 巨大且普遍

所有在线方法的 gap 均超过 370%：
- **S**: RL gap 654.4%, Greedy gap 1694.3%
- **M**: RL gap 434.8%, Greedy gap 424.9%
- **XL**: RL gap 377.6%, Greedy gap 374.9%

即使最优的在线方法，其成本也是离线最优的 **4-17 倍**。

### 2. Gap 反映问题的固有难度

为什么 gap 这么大？
- **离线方法** (ALNS) 拥有完美的未来信息：知道所有任务的到达时间、截止时间、位置
- **在线方法**必须在不确定性下做决策：不知道未来任务的到达，不知道最优的 AGV 分配
- **Gap = 信息不对称的代价**，而非算法能力的不足

### 3. S-Scale: RL gap 远小于 Greedy

- RL gap: 654.4% vs Greedy gap: 1694.3%
- RL 的 gap 仅为 Greedy 的 **38.6%**
- **论点**: 在 S-scale，RL-APC 显著缩小了在线-离线差距

### 4. M/XL-Scale: 两者 gap 相当

- **M**: RL 434.8% vs Greedy 424.9% — 差距仅 10 个百分点
- **XL**: RL 377.6% vs Greedy 374.9% — 差距仅 2.7 个百分点
- **论点**: 在 M/XL，两种方法达到了相似的在线性能天花板

### 5. Gap 随 scale 增大而缩小

- S: ~654-1694%
- M: ~425-435%
- XL: ~375-378%

可能的解释：
- 大规模实例有更多任务和 AGV，统计效应更平滑
- 大规模下"合理决策"的数量增加，随机性影响降低
- 离线优化在大规模下也面临更多约束，gap 自然缩小

## 为什么这个分析重要？

### 对审稿人的意义

1. **校准期望**: 告诉审稿人"不要期望在线方法接近离线最优"
2. **公平评价**: 所有在线方法都面临同样的信息劣势
3. **RL 的价值**: 在 S-scale，RL 把 gap 从 1694% 缩小到 654%（-61%），这是显著的改进
4. **上下文化 M/XL**: M/XL 的 gap 相当说明两者都在做"合理的在线决策"

### 离线方法说明

- **ALNS (Adaptive Large Neighbourhood Search)**: 后验优化器，使用完整问题信息
- **求解模式**: First-Route (FR) — 先路径后充电
- **注意**: ALNS 本身不保证全局最优，只是一个高质量的元启发式上界
- **实际最优 gap 可能更大**: 如果用精确方法（如 MIP），离线解更优，gap 更大

- **Q4 互补**: 即使 gap 相似 (M/XL)，Q4 已经证明 RL 的服务质量更优
