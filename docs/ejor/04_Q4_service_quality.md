# Q4: 成本之外还有什么优势？

> **结论: RL-APC 在服务质量上有压倒性优势——拒绝率极低、完成率更高（M/XL），即使在总成本持平的 scale 也能体现附加价值。**

## 关键发现

### 1. 拒绝率是最大差异

RL-APC 几乎不拒绝任务，Greedy-FR 在中大规模下大量拒绝：
- **S**: RL 0 拒绝 vs Greedy 1.5 拒绝 — 差异较小
- **M**: RL **0 拒绝** vs Greedy **21.8 拒绝** — RL 完全接受所有任务
- **L**: RL **0 拒绝** vs Greedy **16.8 拒绝** — RL 零拒绝但完成率低
- **XL**: RL **1.5 拒绝** vs Greedy **57.7 拒绝** — Greedy 拒绝了大量任务

### 2. 拒绝惩罚成本占 Greedy 总成本的巨大比例

Rejection penalty = 10,000 × rejected_tasks：
- **M**: Greedy 拒绝惩罚 218,000 占总成本 281,349 的 **77.5%**
- **XL**: Greedy 拒绝惩罚 577,333 占总成本 809,830 的 **71.3%**
- RL-APC 的 M-scale 拒绝惩罚为 **0**，XL-scale 仅 14,667

### 3. M-Scale: RL 虽然总成本略高，但服务质量远优

- **成本**: RL 286,665 vs Greedy 281,349 (+1.9%, 统计不显著 p=0.271)
- **完成率**: RL **52.0%** vs Greedy **33.0%** — RL 多完成 57.6% 的任务
- **拒绝率**: RL **0** vs Greedy **21.8** — Greedy 拒绝了大量任务
- **延迟**: RL **21.9** vs Greedy **604.8** — RL 延迟仅为 Greedy 的 3.6%
- **论点**: 以可忽略的成本代价 (+1.9%) 换取了极大的服务质量提升

### 4. XL-Scale: 成本持平但服务质量碾压

- **成本**: RL 814,344 vs Greedy 809,830 (+0.6%, 统计不显著 p=0.655)
- **完成率**: RL **24.7%** vs Greedy **22.3%** — RL 略多
- **拒绝率**: RL **1.5** vs Greedy **57.7** — Greedy 拒绝了 **38 倍**更多的任务
- **论点**: 几乎相同的总成本下，RL 拒绝几乎为零

### 5. S-Scale: RL 成本大胜，服务质量也优

- **成本**: RL 39,846 vs Greedy 94,772 (**-58.0%**)
- **拒绝率**: RL 0 vs Greedy 1.5
- **延迟**: RL 136.7 vs Greedy 17,137 — RL 延迟仅为 Greedy 的 **0.8%**
- **论点**: S-scale 是 RL-APC 的全面胜出场景

### 6. L-Scale: 服务质量叙事复杂

- RL 完成 17.5 任务 / 0 拒绝 vs Greedy 完成 30.4 / 16.8 拒绝
- RL 不拒绝但也不做事（过度保守 → 大量 standby 成本）
- Greedy 主动拒绝不可行任务，完成率反而更高
- **L-scale 是 RL-APC 的已知弱点**

## 成本构成模式

### RL-APC 的成本特征
- **Standby 主导**: RL 的成本主要来自 standby（待命），因为 RL 选择等待而非激进调度
- **几乎零拒绝**: RL 的 rejection penalty 接近 0
- **低 travel/charging**: RL 减少了不必要的移动和充电

### Greedy-FR 的成本特征
- **Rejection 主导 (M/XL)**: Greedy 在 M/XL 的成本主要来自大量拒绝的惩罚
- **Travel/Charging 更高**: Greedy 更积极地移动，导致更高的运营成本
- **Tardiness 更高 (S/L)**: Greedy 在完成任务时延迟更大

## 对论文的意义

这是 RL-APC 最有力的论证之一，特别是对 M 和 XL 两个 scale：

1. **总成本不是唯一指标**: EJOR 审稿人关心实际运营，拒绝率是仓储运营的关键 KPI
2. **M/XL 的叙事**: "在成本几乎相同的情况下，RL-APC 将拒绝率从 21.8/57.7 降至 0/1.5"
3. **成本分解增加可解释性**: 解释了为什么 Greedy 的总成本"看起来不差" — 因为拒绝惩罚被 standby 节省抵消了
4. **L-scale 诚实讨论**: RL 的零拒绝策略在 L 上失效，需要承认并解释
